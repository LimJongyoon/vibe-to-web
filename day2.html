한 세션 = 한 피처 원칙

다른 거 다 까먹어도 이거 하나만

이론

Claude는 모든 대화를 머릿속에 들고 다닙니다. 대화가 길어질수록 더 무거워지고 결국 헷갈리기 시작합니다. 신선한 컨텍스트가 부풀어진 컨텍스트보다 항상 낫습니다.

❌ 자주 하는 실수
> 자기소개 페이지 만들어줘
> 폰트 바꿔줘
> 다크모드 추가해줘
> 취미 섹션도 넣어줘
> 사진 갤러리 만들어줘
> ... (5시간 후)
> 처음에 만든 메인 제목 색깔 검정으로

→ 이 시점에서 Claude는 헛소리를 시작합니다.

✅ 올바른 워크플로우
① 작업 1 시작
   ↓
② 작업 1 완료
   ↓
③ /clear  ← 여기 중요
   ↓
④ 작업 2 시작
   ↓
⑤ 작업 2 완료
   ↓
⑥ /clear → 반복
실습 
[작업 1] 자기소개 페이지 만들기
   → 다 만들고 결과 확인
   → /clear ✨

[작업 2] 취미 갤러리 페이지 추가
   → 다 만들고 결과 확인
   → /clear ✨
💡
/clear는 파일을 지우지 않습니다. 대화 기억만 지우고, 진짜 파일은 그대로 남습니다. 안심하고 자주 쓰세요.

토큰 사용량 확인 — /context

> /context

바(bar) 형태로 시각화됩니다. 80% 이상이면 정리 타이밍.

명령어차이점언제 쓰나
/clear완전 초기화. 대화 다 잊음새 작업 시작 시 (가장 많이 씀)
/compact요약해서 압축. 맥락은 유지같은 작업 계속하되 공간 확보
확인
  • "한 세션 = 한 피처" 원칙을 외웠다
  • /clear가 파일을 안 지운다는 걸 안다
  • /context로 토큰 사용량 확인했다
  • /clear vs /compact 차이를 안다

Plan Mode를 습관처럼

Day 1에서 잠깐 다뤘는데, 오늘 본격적으로 씁니다

이론
모드의미
Plan Mode계획만 세움 (실행 X) — 잘못된 방향을 30초에 잡을 수 있음
Accept Mode계획 + 실행까지 함 (기본)
✅ Plan Mode가 필요한 경우
  • 새 기능을 통째로 추가할 때 (갤러리, 게시판 등)
  • 여러 파일을 동시에 건드리는 작업
  • 30분 이상 걸릴 것 같은 작업
그냥 Accept Mode로 가도 되는 경우
  • 색깔, 폰트 같은 단순 수정
  • 에러 한두 줄 고치기
  • 파일 한 곳만 살짝 만지기
실습

Plan Mode 실전 사용법

  1. 모드 전환
    [Shift + Tab 누름]
  2. 큰 작업 설명
    > 내 자기소개 페이지에 다크/라이트 모드 토글 기능을 추가하고 싶어.
  오른쪽 상단에 버튼 하나 두고, 클릭하면 전체 색상이 부드럽게 전환.
  새로고침해도 마지막 선택이 유지되면 좋겠어.
  3. Claude의 계획 받기
    다음과 같이 진행하려고 합니다:
1. 색상 변수화 — :root와 .light-mode 두 군데에 정의
2. 토글 버튼 추가 — 우상단 fixed, 클릭 시 body에 .light-mode 토글
3. 상태 저장 — localStorage
4. 전환 애니메이션 — CSS transition 0.3s

이대로 진행할까요?
  4. 검토 후 결정
    마음에 들면
    [Shift + Tab → Accept Mode]
> 좋아 진행해줘
    수정 요청
    > 좋은데, localStorage는 빼고
  body 클래스만으로 가자.
💡
습관 팁: 새 채팅 시작할 때 첫 명령어로 Shift+Tab을 누르세요. 큰 작업이면 그대로, 작은 거면 다시 눌러서 Accept Mode로.
확인
  • Plan Mode와 Accept Mode 전환 가능
  • Plan Mode를 써야 할 상황을 안다
  • 실제로 Plan을 받아보고 계획을 읽고 수정 요청해봤다

/memory로 나를 기억시키기

Claude한테 나를 기억시켜봅시다

실습

기억 저장하기

> 앞으로 모든 코드에 한국어 주석을 달아줘. 기억해.
> 내 이름은 임종윤이고, 서강대 아트앤테크놀로지 학생이야. 기억해줘.
💡
"기억해줘"라는 표현이 트리거예요. 이 단어를 쓰면 Claude가 알아서 저장합니다.

기억된 내용 확인하기

> /memory
# Memory
- 사용자 이름: 임종윤
- 소속: 서강대 아트앤테크놀로지
- 선호 폰트: Pretendard
- 선호 컬러: 다크 모드, 오렌지 액센트
- 코드 주석은 항상 한국어

기억 삭제 / 수정

> 내가 Pretendard 폰트 좋아한다는 거 기억에서 지워줘.
  이제 Noto Sans KR을 더 좋아해.

효과 검증 — 새 세션에서 확인

/clear

> 새 페이지 만들어줘. portfolio.html. 내 정보를 담은 포트폴리오 페이지로.
💡
→ 별 말 안 했는데도 "임종윤" 이름, 한국어 주석, 다크 모드로 만들어집니다. 🎉
✅ 저장하면 좋은 것
  • 내 이름, 학과
  • 디자인 취향 (다크/라이트, 색감, 폰트)
  • 코딩 스타일 (네이밍 규칙, 주석 언어)
❌ 저장하지 말 것
  • 비밀번호, API 키, 토큰
  • 개인정보 (전화번호, 주소)
  • 프로젝트 한정 정보 → CLAUDE.md로
확인
  • "기억해"로 정보 저장해봤다
  • /memory로 저장된 내용 확인했다
  • 새 세션에서 효과를 검증했다

멀티 파일로 쪼개기

HTML 하나에 CSS, JS가 다 들어있는 구조를 깔끔하게 정리합니다

이론
❌ 단일 파일의 문제
index.html (300줄)
about.html (300줄)
projects.html (300줄)

색깔 하나 바꾸려면 세 파일 다 수정.

✅ 멀티 파일의 해결
index.html
about.html
projects.html
assets/
  style.css   ← 디자인은 한 곳에
  script.js   ← 동작은 한 곳에

CSS 한 번 고치면 모든 페이지에 반영.

실습

CSS 분리시키기

> 지금 index.html에 들어있는 CSS를 별도 파일로 분리해줘.

요구사항:
- assets/ 폴더 새로 만들고 그 안에 style.css
- index.html에는 <link>로 연결
- CSS 변수(:root)로 색상을 관리하도록 정리
- 한국어 주석 추가

새 페이지 추가하기

> portfolio.html 페이지를 새로 만들어줘.
- 기존 style.css를 동일하게 사용
- 헤더에는 index.html과 동일한 네비게이션
- 본문에는 3개의 프로젝트 카드 (hover 시 살짝 떠오르는 효과)

JavaScript도 분리

> 다크/라이트 토글 기능을 추가하고 싶어.
- assets/script.js 파일에 분리
- 모든 페이지에서 동작하도록
확인

결과 폴더 구조

my-first-web/
├── index.html
├── portfolio.html
├── about.html
└── assets/
    ├── style.css
    └── script.js
  • CSS를 별도 파일로 분리해봤다
  • 새 페이지를 추가하고 같은 CSS를 재사용했다
  • JS도 별도 파일로 분리해봤다

폴더별 CLAUDE.md 전략

토큰 절약 + 정확도 향상

이론

CLAUDE.md가 500줄이 되면? 매 세션마다 그 500줄을 다 읽습니다. 토큰이 줄줄 새요.

❌ 내용 직접 작성
# CLAUDE.md
## API 엔드포인트
- POST /api/auth/login - 로그인 처리...
- GET /api/users/:id - 유저 조회...
- ... (50줄 더)
✅ 참조 링크만
# CLAUDE.md
## 프로젝트 문서
- API 스펙: @docs/api-spec.md
- DB 스키마: @docs/db-schema.md

@파일경로: 관련 작업이 생겼을 때만 읽음.

폴더별 CLAUDE.md

my-first-web/
├── CLAUDE.md             ← 프로젝트 전체 규칙
├── assets/
│   ├── CLAUDE.md         ← assets 폴더 규칙
│   └── style.css
└── pages/
    ├── CLAUDE.md         ← pages 폴더 규칙
    └── about.html

Claude가 pages/about.html을 수정할 때 루트 CLAUDE.md + pages/CLAUDE.md도 같이 읽습니다. 해당 폴더 작업할 때만 그 규칙 적용.

실습 
> CLAUDE.md가 너무 길어졌어. 정리해줘.

방향:
- 루트 CLAUDE.md에는 핵심 원칙과 참조 링크만 (50줄 이내)
- 상세 내용은 docs/ 폴더로 분리
- @ 표기법으로 참조 연결
⚠️
비기너 프로젝트는 단일 CLAUDE.md로 충분합니다. 너무 일찍 최적화하지 마세요. 200줄을 넘을 때 쪼갤 시점입니다.
확인
  • @파일경로 참조 표기법을 안다
  • 폴더별 CLAUDE.md 개념을 이해했다
  • 아직 안 쪼개도 된다는 것도 안다 (조기 최적화 X)

TODO.md로 작업 연속성 유지

"어디까지 했더라?"를 없애는 법

이론

며칠에 걸친 작업에서 컨텍스트를 이어가는 가장 간단한 방법 — TODO.md 파일 하나로 관리합니다.

실습

TODO.md 만들기

# 자기소개 페이지 작업 목록

## 진행 중
- [ ] 다크/라이트 모드 토글 버튼

## 다음 할 일
- [ ] 포트폴리오 페이지 추가
- [ ] 연락처 양식 만들기
- [ ] 반응형 디자인 (모바일)

## 완료
- [x] index.html 기본 구조
- [x] CSS 별도 파일로 분리

## 버그 / 이슈
- 모바일에서 네비 메뉴 깨짐

Claude한테 TODO 관리시키기

> TODO.md 읽고 "다음 할 일"의 첫 번째 항목부터 시작해줘.
> 방금 완료한 작업을 TODO.md의 "완료" 섹션으로 옮겨줘.

세션 종료 시 루틴

> 오늘 작업 마무리할게.
  1. TODO.md를 현재 상태에 맞게 업데이트해줘
  2. 내일 시작할 만한 첫 작업을 "다음 할 일" 맨 위에 추가해줘
  3. 발견한 버그가 있다면 "버그/이슈"에 기록해줘
💡
내일 와서 "TODO.md 읽고 시작해줘"라고 하면 바로 이어서 진행 가능합니다.
확인
  • TODO.md를 만들어봤다
  • Claude한테 TODO 읽고 시작시켰다
  • 세션 종료 시 루틴을 안다

로그를 흘리지 마세요. Claude가 잘못된 가정을 하는 게 보이면, 즉시 Escape로 중단하세요.
실습

에러 보고 — 그대로 붙여넣기

❌ 자기가 해석한 에러
"버튼 클릭이 안 돼"
"뭔가 깨졌어"
✅ 에러 로그 통째로
토글 버튼 추가한 후에 에러 나. 콘솔에 이렇게 떠:

Uncaught TypeError: Cannot read properties of
null (reading 'addEventListener')
    at script.js:12:23

확인해줘.

브라우저 콘솔(F12) → 빨간 글씨를 드래그 복사 → Claude에게 그대로 붙여넣기.

자주 만나는 에러 4가지

① 파일을 못 찾을 때 (404)
GET http://localhost:5500/style.css 404 (Not Found)

→ HTML에서 참조한 CSS 경로가 잘못됨.

② 변수/함수가 없다고 할 때
Uncaught ReferenceError: toggleMode is not defined

→ 함수 이름 오타거나 정의 순서 문제.

③ null/undefined 에러
Uncaught TypeError: Cannot read properties of null

→ 존재하지 않는 요소를 찾으려 함.

④ 문법 에러
Uncaught SyntaxError: Unexpected token

→ 괄호 안 닫힘, 세미콜론 누락 등.

확인
  • Thinking 로그를 읽고 잘못된 가정을 발견해봤다
  • Escape로 잘못된 방향을 중단시켜봤다
  • 에러 메시지를 통째로 복사해서 Claude한테 줬다
  • 4가지 흔한 에러 유형을 안다

실습

├── about.html

단계별 가이드

  1. 준비 — /clear로 깨끗하게 시작

    /memory에 본인 정보 저장 → TODO.md 만들어서 오늘 할 일 정리.

  2. Plan Mode로 설계

    Shift + Tab으로 Plan Mode → "현재 index.html을 멀티 파일 구조로 분리하고 싶다" → 계획 검토 후 Accept Mode.

  3. 멀티 파일 분리

    CSS → assets/style.css / JS → assets/script.js → 결과 확인 → /clear.

  4. 새 페이지 추가 (Skills 활용)

    새-페이지-만들기 Skill 등록 → portfolio.html 추가 → about.html 추가 → 사이사이 /clear.

  5. 마무리

    CLAUDE.md 정비 (300줄 이하) → TODO.md 업데이트 → 폴더 백업.

확인
  • 3개 페이지가 모두 작동한다
  • 페이지 간 네비게이션이 동작한다
  • CSS/JS가 분리되어 있다
  • Skill 하나 이상 등록되어 있다
  • TODO.md가 정리되어 있다
  • CLAUDE.md가 깔끔하다 (300줄 이하)
