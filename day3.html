Git에 올리기

코드의 시간 여행 머신

이론
Git vs GitHub
이름뭐
Git내 컴퓨터에서 돌아가는 버전 관리 프로그램
GitHubGit으로 관리되는 코드를 인터넷에 보관해주는 사이트

Git이 USB라면, GitHub은 클라우드 드라이브.

실습

GitHub 가입

  1. github.com 접속 → Sign up
  2. 이메일, 비밀번호, 아이디 입력

    아이디는 신중하게 — github.com/내아이디/프로젝트 주소로 쓰입니다.

  3. 이메일 인증 → 무료 플랜 선택

Git 설치 확인

git --version
xcode-select --install

git-scm.com/download/win → 다운로드 → Next 계속 (기본값)

# 처음 한 번만 설정
git config --global user.name "본인이름"
git config --global user.email "github이메일"

GitHub에 올리기

  1. desktop.github.com → 설치 → GitHub 로그인
  2. File → Add Local Repository → 본인 폴더 → Create a Repository
  3. Summary에 첫 커밋: 작업 완성 입력 → Commit to main
  4. Publish repository → "Keep this code private" 체크 해제 → Publish
git init
git add .
git commit -m "첫 커밋"
git branch -M main
git remote add origin https://github.com/내아이디/내-프로젝트.git
git push -u origin main
💡
Claude한테 시키는 방법: "이 프로젝트를 GitHub에 처음 올리려고 해. 내 아이디는 ___. 순서대로 알려줘."

이후 업데이트

git add .
git commit -m "수정 내용 설명"
git push
⚠️
비밀번호, API 키가 코드에 있으면 GitHub에 올리지 마세요. Claude한테 .gitignore 만들어달라고 하세요.
확인
  • GitHub 계정 생성
  • 코드 올리고 브라우저에서 저장소 확인

확인

Skills — AI에게 주는 업무 매뉴얼

반복 프롬프트를 재사용 가능한 시스템으로

이론
프롬프트
매번 입력
결과 들쭉날쭉
공유 어려움
Skills
한 번 만들면 자동 호출
품질 일관
Git으로 팀 공유
2단계 로딩 방식

1단계: 스킬 이름 + description만 로드 (항상, ~50-100바이트)
2단계: 실제 호출 시 SKILL.md 본문 전체 로드
평소에는 가볍게, 쓸 때만 전체를 불러오는 구조입니다.

도구평소 공간언제 쓰나
CLAUDE.md전체 내용 매번항상 적용되는 핵심 규칙
Skills설명만 (~50-100B)특정 작업할 때만 필요한 매뉴얼
Sub-Agent에이전트 설명 매번대량 탐색 / 격리된 작업

스킬 폴더 구조

.claude/skills/
└── ppt-generator/            ← 스킬 폴더 이름
    ├── SKILL.md               ← 핵심 파일 (필수)
    ├── template.md            ← 템플릿 (선택)
    └── examples/              ← 예제 (선택)
실습

SKILL.md 작성법

---
name: ppt-generator
description: "PPT 발표자료 자동 생성.
  'PPT 만들어줘', '발표자료 작성', '슬라이드 제작' 요청 시 트리거."
---

## 목적
주제와 핵심 내용을 입력하면 PPT 발표자료를 자동 생성합니다.

## 절차
1. 발표 주제, 대상 청중, 발표 시간 확인
2. 목차 및 슬라이드 구성 설계
3. 각 슬라이드별 내용 작성

## 자체 검증 체크리스트
- [ ] 슬라이드 수가 적절한가?
- [ ] 핵심 메시지가 있는가?
❌ 나쁜 description
description: "문서를 생성하는 스킬"

너무 추상적. Claude가 언제 쓸지 모름.

✅ 좋은 description
description: "PPT 발표자료 자동 생성.
  'PPT 만들어줘', '발표자료 작성' 요청 시 트리거."

핵심 기능 첫 문장 + 트리거 표현 3개 이상.

스킬 만들기

mkdir -p ~/.claude/skills/ppt-generator
# SKILL.md 직접 작성
> /install-plugin skill-creator

설치 후 "스킬 만들어줘"라고 말하면 자동 트리거. description 최적화, frontmatter 설정까지 모범 사례를 자동 적용해서 SKILL.md를 만들어줍니다.

스킬 저장 위치

위치경로적용 대상
개인용~/.claude/skills/<name>/SKILL.md내 모든 프로젝트
프로젝트용.claude/skills/<name>/SKILL.md이 프로젝트만 (Git 공유 가능)
💡
팀 공유: .claude/skills/를 Git에 커밋하면 팀원이 pull 받는 순간 바로 사용 가능합니다.
확인
  • SKILL.md의 frontmatter 구조 (name, description)를 안다
  • description이 트리거 조건이라는 걸 안다
  • Skill 하나 만들어봤다
  • 만든 Skill이 자동 발동되는 걸 확인했다

Sub-Agent & 병렬 실행

메인 Claude 안에서 도우미를 하나 더 띄우는 것

이론
핵심 장점 4가지
  • 병렬 처리 — 여러 작업을 동시에 실행해 전체 소요 시간 단축
  • 컨텍스트 보호 — 조사 결과가 Sub-Agent 쪽에만 남고, 메인에는 요약만 돌아옴
  • 전문화 — 각 Sub-Agent에 전문 역할 부여
  • 재사용 — 한 번 만든 에이전트를 여러 워크플로우에서 활용
✅ Sub-Agent로 분배하기 좋은 것
  • 독립적인 작업을 병렬로 분배
  • 대량 출력이 나오는 작업 격리
  • 명확한 역할과 범위가 있는 작업
❌ 적합하지 않은 것
  • 의존성 있는 작업 무리하게 병렬화
  • 하나의 Sub-Agent에 너무 많은 역할
  • Sub-Agent 간 직접 통신 (불가)
⚠️
Sub-Agent는 다른 Sub-Agent를 생성할 수 없습니다. 메인에서 체인으로 연결하세요.

내장 Sub-Agent 5종

Sub-Agent모델권한용도
ExploreHaiku (빠름)읽기전용코드 탐색, 파일 검색, 구조 파악
Plan상속읽기전용Plan Mode에서 계획 수립 연구
General-purpose상속모든 도구탐색+수정 필요한 복잡한 작업
Bash상속Bash만별도 컨텍스트에서 터미널 명령
Claude Code GuideHaiku읽기전용Claude Code 기능 질문 답변
실습

커스텀 에이전트 만들기 — /agents

  1. /agents 입력
  2. Create new agent 선택
  3. User-level (모든 프로젝트) 또는 Project-level 선택
  4. Generate with Claude → 에이전트 설명 입력
  5. 도구 선택 (Read-only / 전체) → 모델 선택 → 저장
# 에이전트 파일 구조 (YAML + Markdown)
---
name: code-reviewer
description: "코드 리뷰 전문가. 품질, 보안, 모범 사례를 검토."
tools: Read, Grep, Glob, Bash
model: sonnet
---

호출되면:
1. git diff로 최근 변경 확인
2. 수정된 파일에 집중하여 리뷰

피드백 우선순위:
- 🔴 크리티컬 (반드시 수정)
- 🟡 경고 (수정 권장)
- 🟢 제안 (개선 고려)

에이전트 저장 위치

저장 위치어디서 쓸 수 있나
~/.claude/agents/내 모든 프로젝트 (개인용)
.claude/agents/이 프로젝트에서만 (팀 공유)
--agents CLI 플래그지금 이 세션만 (일회용)
 
# CLI에서 즉석으로 에이전트 정의 (일회성)
claude --agents '{
  "quick-reviewer": {
    "description": "빠른 코드 리뷰어",
    "prompt": "변경된 코드를 리뷰하고 핵심 이슈만 보고하세요.",
    "tools": ["Read", "Grep"],
    "model": "haiku"
  }
}'

실전 패턴

대량 출력 격리

테스트 로그처럼 출력이 많은 작업을 Sub-Agent에 위임하고 요약만 받기 → 메인 컨텍스트 보호.

병렬 연구

독립적인 모듈을 각각 별도 Sub-Agent로 동시 분석 → 순차로 30분 걸릴 일이 10분에 끝남.

에이전트 체인

code-reviewer → optimizer 순서로 순차 연결.

확인
  • Sub-Agent와 메인 대화의 차이를 안다
  • 내장 Sub-Agent 5종의 역할을 안다
  • /agents로 커스텀 에이전트 만들어봤다

Hooks — 자동화 엔진

이벤트가 발생하면 자동으로 실행되는 명령어

이론
이벤트 → Hook 감지 → 액션 실행

파일 저장, 도구 호출, 알림 수신 등의 이벤트 발생
→ matcher가 조건 검사 (와일드카드 * 또는 특정 패턴)
→ lint 실행, 포맷 적용, 알림 전송 등 자동 실행

이벤트 타입 4가지

이벤트타이밍설명
PreToolUse도구 호출 직전입력값 검증, 승인/차단/수정 가능
PostToolUse도구 실행 직후결과 검증, 후처리 (린트, 포맷팅 등)
Notification사용자 응답 대기 시알림 전송, 로깅, 외부 서비스 연동
Stop에이전트 턴 종료 시최종 정리, 보고서 생성, 상태 저장
실습

Hook 만들기

> 알림 Hook 만들어줘.
  작업이 끝나면 소리랑 알림 띄워줘.

Claude가 settings.json에 자동 추가해줍니다.

개인용: ~/.claude/settings.json / 프로젝트용: .claude/settings.json

{
  "hooks": {
    "Notification": [
      {
        "matcher": "*",
        "hooks": [
          {
            "type": "command",
            "command": "terminal-notifier -title 'Claude Code' -message '알림이 있습니다' && afplay /System/Library/Sounds/Ping.aiff &"
          }
        ]
      }
    ]
  }
}

matcher: "*" = 모든 상황에 매칭. 특정 도구만 매칭하려면 도구 이름 지정.

⚠️
Hook 실행 중 Claude는 멈춰서 기다립니다. timeout을 꼭 설정하고, 무거운 작업은 백그라운드(&)로 돌리세요.
확인
  • 이벤트 타입 4가지를 안다
  • 알림 Hook 하나 만들어봤다
  • Hook이 실행되는 걸 확인했다

Git Worktree — 병렬 브랜치 작업

하나의 Git 저장소에서 여러 개의 작업 디렉토리를 만드는 기능. 커밋 히스토리는 공유하면서 파일 시스템은 완전히 분리됩니다.

# Claude Code 네이티브 지원
claude --worktree feature-auth
claude -w feature-auth
💡
claude -w 한 줄이면 워크트리 생성 → 브랜치 체크아웃 → 세션 시작까지 자동. 변경 없으면 세션 종료 시 자동 정리.

음성 입력 — /voice

> /voice

Push-to-talk 모드. 스페이스바 누르고 있는 동안 말함, 떼면 변환. 한국어 포함 20개 이상 언어 지원. 복잡한 요구사항을 키보드로 5분 걸릴 프롬프트를 말로 1분에 전달.

MCP 대신 로컬 Bash 스크립트

간단한 작업이라면 MCP보다 로컬 bash 스크립트가 훨씬 가볍습니다. 연결 없이 바로 실행되고 컨텍스트 공간도 절약됩니다.

#!/bin/bash
# scripts/check-db.sh
psql -h localhost -U dev -d mydb -c "SELECT count(*) FROM users;"
> check-db.sh 실행해줘

MCP 연결 불필요. Claude가 Bash로 스크립트를 바로 실행합니다.

커스텀 MCP 서버 빌드

Claude가 작업 중에 자동으로 호출해야 하는 도구가 필요하면 MCP 서버를 만드세요.

> Gemini API를 사용하는 plan-review MCP 서버를 만들어줘.
  review_plan 도구 하나만 있으면 돼.
  plan 텍스트를 받아서 Gemini에 리뷰를 요청하고 결과를 반환하게.

Claude가 MCP 서버 코드를 작성하고 settings.json에 등록까지 도와줍니다.

실습

통합 파이프라인 — 8단계 워크플로우

클라이언트 피드백 10개를 한큐에 처리하는 실전 워크플로우. 수동이면 반나절 → Claude Code 파이프라인으로 30분.

#기능시너지

파워유저 요약

Skills + Sub-Agent + Hooks를 조합하면 1인 개발자가 팀급 생산성을 낼 수 있습니다. 오늘 배운 것들은 각각 독립적으로 쓰이지만, 함께 연결했을 때 진짜 위력이 나옵니다.
