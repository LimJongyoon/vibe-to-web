## Part A — 작업 원칙 ⏱ 60분

### Step 8. 한 세션 = 한 피처 원칙

**[이론]**

Claude는 모든 대화를 머릿속에 들고 다닙니다. 대화가 길어질수록 더 무거워지고 결국 헷갈리기 시작합니다. **신선한 컨텍스트가 부풀어진 컨텍스트보다 항상 낫습니다.**

| ❌ 자주 하는 실수 | ✅ 올바른 워크플로우 |
|----------------|------------------|
| 한 세션에 여러 기능을 계속 추가 → 대화가 길어져 Claude가 헷갈림 | 작업 1 완료 → `/clear` → 작업 2 시작 → `/clear` → 반복 |

💡 `/clear`는 파일을 지우지 않습니다. **대화 기억만 지우고, 파일은 그대로 남습니다.**

**[실습]**

```
[작업 1] 자기소개 페이지 만들기 → 결과 확인 → /clear ✨
[작업 2] 취미 갤러리 추가 → 결과 확인 → /clear ✨
```

토큰 사용량 확인:
```
> /context
```
80% 이상이면 정리 타이밍.

| 명령어 | 차이점 | 언제 |
|--------|--------|------|
| `/clear` | 완전 초기화 | 새 작업 시작 시 (가장 많이 씀) |
| `/compact` | 요약 압축 (맥락 유지) | 같은 작업 계속하되 공간 확보 |

**[확인]**
- [ ] "한 세션 = 한 피처" 원칙을 외웠다
- [ ] `/clear`가 파일을 안 지운다는 걸 안다
- [ ] `/context`로 토큰 사용량 확인했다

### Step 9. Plan Mode를 습관처럼

**[이론]**

| 모드 | 의미 |
|------|------|
| **Plan Mode** | 계획만 세움 (실행 X) — 잘못된 방향을 30초에 잡을 수 있음 |
| **Accept Mode** | 계획 + 실행까지 함 (기본) |

Plan Mode가 필요한 경우:
- 새 기능을 통째로 추가할 때
- 여러 파일을 동시에 건드리는 작업
- 30분 이상 걸릴 것 같은 작업

**[실습]**

1. `Shift + Tab` → Plan Mode 진입
2. 큰 작업 설명:
   ```
   > 내 자기소개 페이지에 다크/라이트 모드 토글 기능을 추가하고 싶어.
     오른쪽 상단에 버튼 하나 두고, 클릭하면 전체 색상이 부드럽게 전환.
   ```
3. Claude의 계획을 받은 후 검토
4. 마음에 들면: `Shift + Tab` → Accept Mode → "좋아 진행해줘"
5. 수정 필요하면: "localStorage는 빼고 body 클래스만으로 가자."

💡 습관 팁: 새 채팅 시작할 때 첫 명령어로 `Shift+Tab`. 큰 작업이면 그대로, 작은 거면 다시 눌러서 Accept Mode로.

**[확인]**
- [ ] Plan Mode와 Accept Mode 전환 가능
- [ ] 실제로 Plan을 받아보고 계획을 읽고 수정 요청해봤다

### Step 16. 에러와 친구되기

**[이론]**

Thinking 로그를 읽으세요:
```
● 기존 CSS를 보니 색상이 :root 변수로 정의...
  ...React로 만들면 깔끔하겠네요. ← 잘못된 가정!
  [Escape로 즉시 중단]
```
30초 만에 잡음. 방치하면 30분 낭비.

**[실습]**

에러 보고 — 통째로:
```
토글 버튼 추가한 후 에러 나. 콘솔에 이렇게 떠:

Uncaught TypeError: Cannot read properties of
null (reading 'addEventListener')
    at script.js:12:23

확인해줘.
```

자주 만나는 에러:
- `404 Not Found` — HTML에서 참조한 경로가 잘못됨
- `ReferenceError: X is not defined` — 함수 이름 오타 or 정의 순서 문제
- `TypeError: Cannot read properties of null` — 존재하지 않는 요소 접근
- `SyntaxError: Unexpected token` — 괄호 안 닫힘, 세미콜론 누락

**[확인]**
- [ ] Thinking 로그를 읽고 Escape로 중단시켜봤다
- [ ] 에러 메시지를 통째로 복사해서 Claude한테 줬다

## 치트시트

**컨텍스트 관리:**
```
/clear     대화 초기화 (새 작업 시작 시)
/compact   대화 압축 (맥락 유지)
/context   토큰 사용량 확인
/memory    개인 메모리 열기
/skills    설치된 Skills 목록
```

**워크플로우 6원칙:**
```
1. 한 세션 = 한 피처     끝나면 /clear
2. Plan Mode 먼저        큰 작업은 무조건
3. 작은 변경 → 확인 → 반복
4. 에러는 그대로 던지기
5. Thinking 로그 읽기    잘못된 가정이면 Esc
6. /context 자주 확인    80% 넘으면 정리
```

**SKILL.md 템플릿:**
```yaml
---
name: 영-소문자-하이픈
description: 사용자가 ~할 때 사용. ~을 수행한다.
---

# 제목
## 단계
1. 첫 번째
2. 두 번째
```

**막혔을 때:**

| 증상 | 처방 |
|------|------|
| Claude가 헷갈려함 | `/clear` 후 재시작 |
| 잘못된 방향 같아요 | `Esc` 즉시 중단 |
| 어디까지 했는지 모름 | TODO.md 확인 |
| 화면 깨졌어요 | `Esc Esc`로 복원 |
| 터미널 닫아버렸어요 | `claude` 실행 후 `/resume` |
